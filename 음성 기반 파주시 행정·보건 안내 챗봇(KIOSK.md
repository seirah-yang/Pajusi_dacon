# 프로젝트 구조: 
이 프로젝트는 어르신·독거노인을 위한 음성 기반 민원·보건 안내 서비스로, 파주시에서 보유한 문서와 조직 정보를 AI가 음성으로 안내하는 키오스크형 시스템입니다

### STT(음성 인식)

- 시민이 말한 음성을 Whisper로 텍스트로 변환합니다.

### RAG(문서 검색)

- 변환된 문장을 기반으로 파주시 보건소·행정 조직 데이터(JSON)와 조례 문서(BM25/FAISS)에서 가장 관련 있는 문장을 검색합니다.

### GPT 답변 생성

- 검색된 문서 내용을 토대로 SKT A.X(AX4) 모델이 쉬운 말·노인 친화형 답변을 생성합니다.

### TTS(음성 출력)

- 생성된 답변을 gTTS로 음성(MP3)으로 만들어 키오스크 화면에서 바로 재생합니다.

🔷 서비스 특징

 - 24시간 음성 민원 안내(키오스크·앱 가능)
 
 - 노인 친화(UI + 말투) 및 단순 문장 출력
 
 - 파주시 조직·전화번호·담당자 기반 정확 안내
 
 - 무음 감지 자동 종료로 어르신 사용성 강화
 
 - 행정문서 + 보건 조직 데이터 기반 RAG

🔷 적용 가능 분야

 - 보건소·읍면동 민원실 키오스크
 
 - 독거노인 돌봄센터
 
 - 공공기관 음성 챗봇
 
 - AI 기반 민원 자동응답 시스템

---------------------------
## 1) 음성 인식(STT) — stt_tts.py

 Whisper 및 GPT STT 모델을 활용하여 사용자의 음성을 텍스트로 변환
 
 노년층의 말투·발음 인식 문제를 보완하기 위해 가벼운 전처리 + 반복 음절 정규화 적용

## 2) 문서 검색(RAG) — rag_engine.py / rag_service.py

 파주시 조례·행정문서(JSONL) + 보건소 조직 정보(JSON)를 통합하여 BM25 + FAISS Hybrid 검색

 노년층 문의(“어디 전화해야 해요?” / “어디 가면 돼요?”)에 대비하여 전화번호·담당업무·부서명 우선 검색 알고리즘 구성

 검색된 문서(근거)를 GPT가 사용할 수 있는 단일 Text Context로 변환

## 3) 답변 생성(GPT) — llm_engine.py / paju_careon_fastapi.py

 SKT A.X(AX4) 모델을 활용하여 “노인 친화형·짧은 문장·쉬운 단어” 원칙으로 답변 생성

 조례·행정근거가 있으면 반드시 반영하도록 프롬프트 설계

 정확한 정보가 없을 때는 민원콜센터 연결 원칙 적용 → 안전성 확보

## 4) 음성 출력(TTS) — stt_tts.py

 gTTS 기반 MP3 생성

 키오스크 UI에서 즉시 재생되도록 /static/tts/ 경로에 저장

 괄호 안 내용 제거(조문번호 등)로 어르신 발화 이해도 향상

## 5) 웹 서비스(FastAPI + UI) — paju_careon_fastapi.py / index.html

  키오스크 UI는 자동 무음 감지로 손쉬운 음성 입력 지원

 STT, RAG, GPT, TTS 처리 후 텍스트 + 음성 안내를 동시에 제공
